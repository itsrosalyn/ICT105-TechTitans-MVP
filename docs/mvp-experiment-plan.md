# MVP Experiment Plan

## 1. Group and Project Information

- **Group Name:** Tech Titans
- **Project Title:** QR Code Attendance System with GPS Verification
- **Repository Link:** https://github.com/itsrosalyn/ICT105-TechTitans-MVP
- **Main Target User:** ICT students and lecturers at Rangsit University
- **Prototype Platform:** Web-based clickable prototype (HTML/CSS/JavaScript)

---

## 2. Experiment Objective

> We want to test whether students can successfully complete the QR code attendance process, including GPS location verification, and whether lecturers can easily manage attendance records using the prototype without assistance.

---

## 3. Requirement Scope for the Experiment

| Requirement ID | Requirement Summary | Related Screen/Feature | Tested in This Experiment? |
|---|---|---|---|
| FR-01 | Homepage / Login Screen | Login Page | Yes |
| FR-02 | Primary user pathway | Student attendance workflow | Yes |

| FR-06 | Search and filter | Attendance history search | Yes |

---

## 4. MVP Experiment Type

**Selected experiment type:**
- Clickable prototype
- Simple web prototype

**Reason for selection:**

A clickable web prototype allows users to complete the main attendance workflow, including login, QR code scanning, GPS verification, and attendance tracking. It enables the team to evaluate usability, identify interface issues, and collect user feedback before implementing the complete backend system.

---

## 5. Test Users

| Test User Group | Number of Testers | Why They Are Relevant |
|---|---:|---|
| ICT Students | 5 | Primary users who will use the attendance system in class. |
| ICT Lecturers | 2 | Responsible for creating attendance sessions and monitoring attendance records. |

---

## 6. Experiment Procedure Summary

Participants will be given the prototype and asked to complete a series of tasks without assistance. Students will log in, scan a QR code, allow GPS verification, receive an attendance confirmation, and review their attendance history. Lecturers will generate a QR code, create an attendance session, and view the attendance dashboard. The team will observe task completion, note usability issues, record participant feedback, and measure whether users can complete the workflow successfully.

---

## 7. Expected Learning

The team wants to determine whether users can complete the attendance process easily and whether the QR code and GPS verification workflow is understandable and effective. Based on the results, the team will decide whether to proceed with the current prototype, improve the user interface, or revise specific features before further development.
