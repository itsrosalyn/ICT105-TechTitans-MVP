# Business Model Canvas

## Project Title

QR Code Attendance System with GPS Verification

## 1. Customer Segments

**Primary Customers:**
- ICT students at Rangsit University
- ICT lecturers at Rangsit University

**Secondary Customers:**
- University administrators
- Academic departments responsible for attendance management

## 2. Value Proposition

The QR Code Attendance System with GPS Verification provides a faster, more accurate, and secure way to manage class attendance. It reduces lecture time spent on attendance, prevents proxy attendance by verifying students' locations, minimizes manual administrative work, and allows both students and lecturers to monitor attendance records in real time.

## 3. Channels

- University web application
- Mobile web browser on students' smartphones
- QR codes displayed during class
- University LMS or class communication channels
- GitHub prototype demonstration

## 4. Customer Relationships

- Secure student and lecturer login
- Self-service attendance check-in
- Instant attendance confirmation
- Real-time attendance history for students
- Lecturer dashboard for attendance monitoring
- User feedback for continuous improvement

## 5. Revenue Streams / Sustainability Logic

This project is designed as an institutional solution rather than a commercial product. Its value comes from:
- Reducing administrative workload
- Saving lecture time
- Improving attendance accuracy
- Supporting digital transformation within the university

If expanded in the future, the system could be licensed or customized for other educational institutions.

## 6. Key Resources

- Development team
- HTML, CSS, JavaScript
- Firebase or MySQL database
- QR code generation and scanning library
- GPS/Geolocation API
- GitHub repository
- Wireframe and prototype design tools (e.g., Figma)

## 7. Key Activities

- Design the user interface and wireframes
- Develop the attendance system prototype
- Generate unique QR codes for each class session
- Verify GPS location during attendance check-in
- Store and manage attendance records
- Test the prototype with users
- Improve the system based on feedback

## 8. Key Partners

- Rangsit University
- ICT lecturers
- ICT students
- Course instructor
- Firebase or cloud hosting provider
- GitHub

## 9. Cost Structure

- Development time
- Internet and cloud hosting services
- Firebase or database hosting
- UI/UX design tools
- Testing and maintenance
- Project documentation and presentation preparation

## Final Note

This business model directly supports the MVP by focusing on the essential features identified through customer discovery. The prototype includes role-based login, QR code attendance, GPS location verification, attendance confirmation, attendance history, and a lecturer dashboard. These features solve the main user problems of proxy attendance, manual attendance management, and inefficient attendance tracking while remaining practical and achievable within the project scope.
