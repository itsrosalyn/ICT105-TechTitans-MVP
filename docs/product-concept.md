# Product Concept

## 1. Project Title

QR Code Attendance System with GPS Verification

## 2. Target User

**Primary users:** ICT students who attend lectures and record their attendance using QR codes.

**Secondary users:** ICT lecturers who create attendance sessions, monitor attendance records, and manage student attendance.

## 3. Problem Statement

Students and lecturers need a more reliable attendance management system because current attendance methods—including QR-code attendance, paper sign-in sheets, verbal roll calls, and spreadsheets—do not consistently verify physical presence, allow proxy attendance, and require manual attendance management.

## 4. Proposed IT Product

A web-based attendance management system that enables lecturers to generate a unique QR code for each class session. Students scan the QR code to check in, while GPS verification confirms they are physically present in the classroom. The system records attendance in real time and provides attendance history and monitoring through a lecturer dashboard.

## 5. Core Value Proposition

The system provides a faster, more accurate, and secure attendance process by reducing lecture time spent on attendance, preventing proxy attendance through GPS verification, and allowing both students and lecturers to monitor attendance records in real time.

## 6. MVP Scope

The MVP will focus on:

* Student login
* Lecturer login
* QR code generation for attendance sessions
* QR code scanning for attendance check-in
* GPS-based location verification
* Attendance confirmation after check-in
* Student attendance history
* Lecturer dashboard with real-time attendance records

## 7. Out of Scope for Current Semester

* Face recognition for attendance verification
* Integration with the university Learning Management System (LMS)
* Push notifications and email reminders
* Advanced attendance analytics and reporting
* Native mobile application deployment
* Biometric authentication
