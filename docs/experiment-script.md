# Experiment Script

## 1. Opening Script

Thank you for helping us test our QR Code Attendance System prototype. This is **not** a test of you. We are testing whether our product idea and interface are easy to understand and use. Please think aloud while completing each task. Feel free to share your thoughts, questions, or any difficulties you experience.

---

## 2. Test Scenario

### Student Scenario

You are an ICT student attending a lecture at Rangsit University. Your lecturer asks you to record your attendance using the QR Code Attendance System. You need to log in, scan the QR code, verify your location, confirm your attendance, and check your attendance history.

### Lecturer Scenario

You are an ICT lecturer preparing for class. You need to log in, generate a QR code for today's attendance session, and monitor student attendance using the dashboard.

---

## 3. Tasks for Tester

| Task No. | Task Instruction | Related Requirement | Expected Result |
|---|---|---|---|
| T1 | Log in as a student. | FR-01, FR-02 | Student successfully accesses the system. |
| T2 | Scan the QR code to record attendance. | FR-03 | QR code is scanned successfully. |
| T3 | Allow GPS location verification. | FR-03, FR-10 | System verifies the student's location. |
| T4 | Confirm that attendance has been recorded. | FR-11 | Student receives a success confirmation message. |
| T5 | View attendance history and search for a previous attendance record. | FR-05, FR-06, FR-07, FR-08 | Student can view and search attendance records successfully. |
| T6 | Log in as a lecturer. | FR-01, FR-02 | Lecturer successfully accesses the dashboard. |
| T7 | Generate a QR code for a new attendance session. | FR-09 | QR code is generated successfully. |
| T8 | View the attendance dashboard and attendance summary. | FR-05, FR-09, FR-12 | Lecturer can monitor attendance records and summary information. |

---

## 4. Observation Notes

Record:
- Where the tester hesitates.
- Any incorrect clicks or navigation mistakes.
- Questions asked during the test.
- Any confusing screens or unclear instructions.
- Suggestions for improving the interface or workflow.
- Time taken to complete each task.
- Whether each task was completed successfully without assistance.

---

## 5. Closing Questions

### For Students

- Was the attendance process easy to understand?
- Was scanning the QR code and verifying your location straightforward?
- Was the attendance confirmation message clear?
- Was it easy to find your attendance history?
- What would you improve about the system?

### For Lecturers

- Was it easy to generate a QR code and start an attendance session? If not, what was difficult?
- Did the attendance dashboard provide all the information you needed? If not, what additional information would you like to see?
- If this system were available for your classes, would you use it instead of your current attendance method? Why or why not?
